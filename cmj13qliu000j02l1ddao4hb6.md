# ✅ Next.js を使うべきタイミング

## 1. 🔍 SEO が重要なとき（SSR/SSG が必要）

ブログ、ランディングページ、EC、ニュースサイトなど  
→ Next.js はサーバーサイドで事前にレンダリングでき、Google / 検索エンジンに内容を正しく読ませられる。

## 2. 🖥️ プロジェクト内でサーバーサイド処理が必要なとき

例：

- API ルート（ログイン、Webhook、アップロードなど）
- Server Actions（ミューテーション処理）
- 認証（Auth.js）
- クライアントとサーバーのコードを同じリポジトリで扱いたい

→ Next.js はフルスタック構成を素早く作れる。

## 3. 🧭 強力なルーティングが欲しいとき
Next.js に標準で含まれるもの：
- ファイルベースのルーティング
- ネストされたレイアウト
- ローディング / エラー境界
- ミドルウェア  

→ ボイラープレートが大幅に減る。

## 4. 🚀 パフォーマンス最適化を自動でやりたいとき
Next.js は次を自動で対応：
- 画像最適化
- スクリプト最適化
- スマートキャッシュ
- ルートプリフェッチ
- 動的・静的レンダリングの切り替え

# ❌ Next.js を使わないほうが良いケース

## 1. 🙅‍♂️ SEO 不要で、完全にクライアントサイドの SPA の場合
例：
- ウェブ上で実行されるアプリ
- 社内ツール
- 非公開ダッシュボード
- ブラウザだけで完結するアプリ（ゲームなど）

→ React + Vite のほうが軽くてシンプル。

## 2. 🪶 軽量なマイクロフロントエンドを作りたいとき
→ Vite や小さめのフレームワークが適切。
